# Null//Drift — Status snapshot · 2026-05-15 5:15 PM

Port of `~/code/hacking-game` (Phaser) → `~/code/nulldrift-godot` (Godot 4.6 3D iso).
Goal: realtime emissive bloom lighting that Phaser couldn't produce.

## What ships right now

### Scenes (3 / 53)
- **Title** (`scenes/title.tscn`) — black starfield, neon-tube **NULL//DRIFT** logo
  with pink frame, menu (NEW GAME · CONTROLS · OPTIONS · SOUNDTRACK), rain,
  music. Direct port of `IntroSequence.vue`'s Three.js scene (capsule meshes
  replace Three.js CapsuleGeometry, Godot glow replaces UnrealBloomPass).
- **Apartment** (`scenes/apartment.tscn`) — iso 3D room, pixel-art player
  billboard, real ceiling-lamp shadows, emissive window panel (purple
  night-city view), green CRT terminal, pink neon LED strip under desk,
  cat NPC, door interactable that exits to city.
- **City** (`scenes/city.tscn`) — outdoor iso block: 12 buildings with
  procedural emissive windows, 5 neon signs (pink/cyan/purple/orange),
  6 streetlights with downward spot, crossroads with road + sidewalks,
  apartment door for re-entry. Currently too dark to read building bodies;
  needs ambient lift or more streetlight coverage.

### Renders (verify visually at `_renders/`)
- `title.png` — looks like the real Phaser title screen
- `apartment.png` — pixel player visible, window glow, CRT spill, ceiling-light shadow
- `city.png` — only emissive elements visible (needs lighting pass)

## Architecture

**Data-driven. Scenes are thin glue. New scenes = new dicts, not new code.**

```
res://
├── audio/music/         48 .mp3 tracks from ~/code/music/nulldrift/
├── assets/
│   ├── sprites/         player-hacker.png (3×4 walk grid)
│   └── textures/        title-bg.png (kept for reference)
├── data/                ← pure data (Aaron's "global libraries" ask)
│   ├── colors.gd        Palette: NEON_GREEN, NEON_CYAN, NEON_PINK, etc.
│   ├── story_flags.gd   StoryFlags: HAS_CYBERDECK, MET_NYX, etc.
│   ├── menus.gd         Menus: TITLE + PAUSE items, title_items(has_save)
│   ├── npcs.gd          NPCs: cat, Nyx (diner + city), Tony — with conditions
│   ├── dialogue.gd      Dialogue: branching trees, condition-matched
│   ├── quests.gd        Quests: wakeUp, atmWitness, firstHack, dinerMeeting
│   ├── items.gd         Items: cyberDeck, pizzaSlice, creditChip, whip
│   └── scenes.gd        Scenes: TITLE, APARTMENT, CITY (declarative)
├── scripts/
│   ├── systems/         ← global libraries (autoloads + static helpers)
│   │   ├── game_state.gd      autoload — flags, quests, inventory, credits
│   │   ├── save_manager.gd    autoload — JSON save to user://save.json
│   │   ├── music.gd           autoload — 8 categories × random crossfade
│   │   ├── interaction.gd     autoload — global prompt + E-key zones
│   │   ├── dialogue_overlay.gd autoload — global dialog box
│   │   ├── scene_builder.gd   static — env, camera, room, furniture,
│   │   │                       outdoor (ground/roads/buildings/signs/streetlights)
│   │   └── npc_spawner.gd     static — NPCs.for_scene → live nodes
│   ├── scenes/
│   │   ├── title.gd           thin glue — neon logo geometry + Menus
│   │   ├── apartment.gd       thin glue — _on_interacted routes actions
│   │   └── city.gd            thin glue — outdoor variant
│   └── capture.gd       offscreen screenshot tool for verification
├── scenes/              .tscn files (just wrap a Node3D + script ref)
└── _renders/            screenshot outputs from capture.gd
```

**Adding a new scene now = ~30 lines of code:** new dict in `scenes.gd`,
new NPCs/dialogue if any, new `scripts/scenes/<name>.gd` mirroring city.gd,
new `.tscn` that wraps it.

## Known issues / TODO before declaring city done

1. **City too dark** — only windows + neon signs visible. Bumped ambient
   to 0.7 but building bodies still read as black. Either lift ambient
   higher, brighten building albedo, or add a soft fill light. Streetlight
   spots may not be hitting the ground at the iso camera angle.
2. **Apartment cat NPC** — spawned at (-3, 0.4, 1.5) but not visible in
   the current camera frame. Either reposition or widen camera.
3. **Title letters** — N looks like /\\, R looks like ['; faithful to the
   Three.js tube geometry but readability marginal. Could replace with
   a custom font / SDF later.
4. **Interactables fire on intersection** — Apartment door action routes
   correctly but apartment→city→apartment works without a "you've already
   been there" gate. Use `GameState.flags["leftApartment"]` flag check.
5. **Quest manager** integration — quests auto-start on GameState._ready
   but no quest objectives wire to actual gameplay yet (no Nyx dialogue
   completes `dinerMeeting`, no ATM event triggers `atmEventDone`).

## How to run

```bash
godot --path ~/code/nulldrift-godot
# or open Godot editor on that folder
```

Capture a still of any scene:
```bash
godot --path . res://scenes/_capture.tscn -- res://scenes/<name>.tscn <out_name>
# PNG appears at _renders/<out_name>.png
```

## Music library

48 tracks from `~/code/music/nulldrift/` are bundled. Categories drive
selection per scene (`Music.play_category("apartment")` picks a random
track from its bucket and crossfades). Edit categories at
`scripts/systems/music.gd::CATEGORIES`.
